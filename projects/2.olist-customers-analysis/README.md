巴西电商客户画像分析项目
# 项目简介
本项目是一个基于 Hadoop 生态系统的端到端数据分析解决方案。项目以巴西知名电商平台 Olist 的公开数据集为基础，聚焦于客户维度，旨在通过标准化的数据仓库构建流程（ETL）和商业智能（BI）可视化，深入挖掘巴西电商市场的客户地理分布、消费行为及复购特征。
项目完整演示了从原始数据加载、数据清洗（ETL）、数据仓库分层建模（ODS -> DWD -> DWS）、多维分析（HiveQL）到最终可视化报表（FineBI）的全链路开发过程，适合作为大数据离线分析项目的实战参考。
# 技术栈
- 计算引擎：Apache Hive
- 数据存储：HDFS
- 数据可视化：FineBI
- 开发语言：SQL (HiveQL)
- 数据来源：[Olist Brazilian E-Commerce Public Dataset (Kaggle)](https://www.kaggle.com/datasets/olistbr/marketing-funnel-olist)
# 项目架构与流程
本项目遵循经典的数据仓库分层架构，具体流程如下：
1. 数据准备与加载
- 创建数据库及原始数据表。
- 将 CSV 源数据加载至 Hive 表中。
2. 数据清洗与ETL
- 去噪：去除 CSV 导入时可能产生的引号。
- 去重：基于 customer_unique_id 识别唯一客户，解决同一客户多订单 ID 的问题。
- 标准化：统一城市名称大小写，过滤空值与无效行。
3. 多维数据分析
- 利用 HiveQL 进行聚合查询，计算关键指标（如客户分布、复购率、热门城市）。
- 将分析结果持久化存储为结果表，供 BI 工具调用。
4. 可视化报表设计
- 基于分析结果表，在 FineBI 中设计交互式仪表板，直观展示业务洞察。
# 数据仓库模型
本项目设计了清晰的表结构以支持分析需求：

| 表名 | 层级 | 描述 |
| :--- | :--- | :--- |
| olist_customers | ODS | 原始客户数据表，直接映射源文件 |
| tb_customers_cleaned | DWD | 明细数据层，经过清洗、去噪、标准化的宽表 |
| tb_unique_customers | DWS | 服务数据层，以客户为粒度的聚合表，用于计算复购率 |
| tb_rs_* | ADS | 应用数据层，针对具体报表需求生成的结果集 |

# 核心分析指标
项目围绕客户画像实现了以下五大核心分析主题：
1. 地理分布分析
- 州级分布：统计各州（State）的客户总量与占比，识别核心市场区域（如圣保罗州 SP）。
- 城市排名：筛选客户数量最多的 TOP 20 城市，辅助精准营销投放。
2. 客户价值分析
- 复购率分析：基于 customer_unique_id 识别重复购买行为，将客户划分为“单次购买”、“2次购买”、“3-5次购买”等群体，评估用户粘性。
3. 基础统计概览
- 统计总订单记录数、唯一客户数、覆盖的州数及城市数，建立宏观业务认知。
